# ArkGate 4.0.1

- **Prepared:** 2026-07-24
- **npm:** `arkgate@4.0.1` (publish pending — flip to published only after `npm view arkgate@4.0.1` and `dist-tags.latest`)
- **Git tag:** `v4.0.1` (after merge)
- **Status:** prepared

Patch over **4.0.0**. Field footgun fix for stale global CLI on upgrade, ArkRules HTML/doctor
catalog honesty, dual-plane product wording, and internal coverage attribution.
No required config migration. Does not weaken write gate or CI. Does not close Z09 / RB-11 field claims.

## What changed

| Area | 4.0.1 result |
|------|--------------|
| Stale global CLI | `ark upgrade` / `ark update` fail closed when PATH CLI is outside project pin and older; prefers `npx`/`pnpm exec`/`yarn` + `arkgate` recovery; `--json` refuse payload |
| `/ark-upgrade` skill | Resolves project-local CLI first; aborts on old PATH-only binary |
| HTML / doctor ArkRules | `rulesUnderContract` lists per-layer sensors, uncovered, covered sample (not counts-only); catalog truncation flags |
| HTML advisories | Doctor advisory sections use `class="section card"` |
| Dual-plane honesty | Product surfaces state structure vs invariants never merge into one architecture score |
| Coverage floors (internal) | Branch threshold 83.0 → **82.5**; ESM imports so V8 attributes upgrade/HTML tests |

## What did not change

- Write gate / CI fail-closed semantics
- ArkRules opt-in semantics (no `arkRules` → same inter-layer verdict as 4.0.0 / 3.9.x)
- Host hardness claims
- Z09 / RB-11 field claims (still open)
- Y07 / Y09 ROADMAP status (still **parked**)
- Package budget ceilings (still 4.0.0 clean-candidate headroom)

## Upgrade

```bash
npm install -D arkgate@4.0.1
# or: npm install -D arkgate@latest   # after this patch is latest
npx arkgate-check --install-agent-gates --force
npx arkgate-check --doctor --json
# prefer project-local upgrade:
npx arkgate upgrade
# or: pnpm exec arkgate upgrade
```

No config migration. Optional: refresh gates/skills after upgrade. Prefer project-local
`npx arkgate` / `pnpm exec arkgate` over a global Homebrew/npm `ark` when upgrading.

## Security notes (honest)

No dependency CVE bumps in this patch. The stale-CLI guard fails closed on mutative upgrade
when the running binary is outside the project pin and older than the installed package; it
does not change architecture gate verdicts.
